# 长期记忆

## 项目架构决策

### 物资区域存储模型（2026-05-13 确定）
**每个区域的同名物资是独立的数据库记录，互不干扰。**

- 同名物资在不同区域 = 多条独立记录（各有自己的 `数量`、`存放区域` 字段）
- 入库逻辑：按 `物资名称` + `存放区域` 查找记录，找到则叠加数量，没找到则新建
- 不再使用 `数量_A区`、`数量_B区` 等字段（这些字段会被入库逻辑跳过）
- 导出时每条记录独立，数量不会合并

### 数据库表结构
- `materials`: id, image, custom_fields(JSON), created_at
- `custom_fields` 关键字段：物资名称、数量、存放区域
