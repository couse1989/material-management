# 2026-05-13 工作记录

## 重构：出入库改为按区域独立记录

**问题：**
当前设计用同一条物资记录的 `数量_A区`、`数量_B区` 字段区分区域，导致：
1. 导出时数量合并
2. 跨区域操作时数据互相干扰

**解决方案：**
每个区域的同名物资是独立的数据库记录，互不干扰。

**修改内容：**

### 后端 `backend/app.py` - `inbound_material`
- 按 `物资名称` + `存放区域` 查找已有记录
- 找到 → 叠加数量；未找到 → 以选中物资为模板新建记录
- 跳过所有 `数量_*` 字段，避免历史脏数据干扰
- 日志关联到实际操作的记录 ID

### 前端 `Inbound.vue`
- 移除 `onMaterialChange` 自动填充区域逻辑（用户手动选择）
- 简化 `groupedMaterials`：按 `存放区域` 分组，同名物资去重
- 移除 `hasAreaData`、`getAreaQuantity`、`getAreaQuantityDisplay` 方法

### 前端 `Outbound.vue`
- 简化 `groupedMaterials`：每条记录已对应一个区域，直接按 `存放区域` 分组
- 移除 `hasAreaData`、`getAreaQuantity` 方法

### 前端 `Inventory.vue`
- 简化 `getAreaQuantities`：直接读取 `存放区域` + `数量` 字段
- 不再解析 `数量_*` 字段

**已推送：** commit `44c0b8f` → GitHub main 分支

## 修复：会话过期后自动注销

**问题：**
登录后一段时间，session 过期，访问页面时提示"加载物资失败"，需手动注销再登录。

**原因：**
- 前端路由守卫只检查 `localStorage.getItem('user')`，不会随服务端 session 过期而清除
- 各页面各自处理 API 错误，无全局 401 处理

**修复：** `frontend/src/main.js`
- 添加 Axios 响应拦截器 `axios.interceptors.response.use`
- 捕获 401 响应 → 清除 localStorage → `ElMessage.warning` 提示 → 跳转 `/login`
- 排除 `/api/login` 请求（登录失败本身返回 401，不应重定向）
- 用 `isRedirecting` 标志防止多次弹窗/重定向

**已推送：** commit `bc17a84` → GitHub main 分支

## 修复：登录后侧边栏菜单不显示

**问题：**
首次登录后，侧边栏（入库、出库、字段管理等子菜单）不显示，需手动刷新页面才正常。

**原因：**
- `App.vue` 中的 `isAuthenticated`、`isAdmin`、`currentUser` 是计算属性
- 计算属性读取 `localStorage`，但 `localStorage` **不是 Vue 的响应式数据**
- Vue 无法追踪 `localStorage` 的变化，登录后计算属性不重新求值
- `Login.vue` 调用 `this.$router.push('/')` 后，`App.vue` 不会重新渲染，`isAuthenticated` 仍为 `false`
- 手动刷新后组件重新挂载，计算属性重新执行，读到正确值

**修复：** `frontend/src/App.vue`
- 将 `isAuthenticated`、`isAdmin`、`currentUser` 从 `computed` 改为 `data()` 属性（初始化时从 `localStorage` 读取）
- 新增 `syncAuthState()` 方法：从 `localStorage` 同步最新状态到 `this.isAuthenticated/isAdmin/currentUser`
- 添加 `watch: { '$route' }`，路由变化时调用 `syncAuthState()`
- `handleLogout()` 中同步清除 `this.isAuthenticated/isAdmin/currentUser`

**已推送：** commit `85325cb` → GitHub main 分支

## 修复：登录失败记入登录日志 + 删除物资补充操作日志

**问题1：登录失败信息不显示在登录日志**
- 成功登录记在 `login_logs`，失败登录记在 `operation_logs`，用户希望在登录日志里看到失败记录

**修复：** `backend/app.py`
- `login_logs` 表新增 `status` 字段（迁移逻辑放在 `init_db()`）
- 成功登录：`INSERT INTO login_logs (username, ip_address, status) VALUES (?, ?, 'success')`
- 失败登录：改为写入 `login_logs (username, ip_address, status) VALUES (?, ?, 'failed')`，不再写 `operation_logs`
- `get_login_logs()`：SELECT 显式指定字段，包含 `status`

**问题2：删除物资没有操作日志**
- `delete_materials_batch()` 和 `delete_material()` 均未记录日志

**修复：** `backend/app.py`
- 删除前先查询物资名称（`custom_fields->物资名称`）
- 删除前写入 `operation_logs：(operation_type='删除', material_id, material_name, operator, remark)`
- 再执行 DELETE

**前端适配：** `frontend/src/views/Logs.vue`
- 登录日志表格/卡片：新增「状态」列，用 `el-tag` 显示（成功=绿色，失败=红色）
- 操作日志：`getOperationTypeTag/Text` 补充 `'删除':'danger'/'删除物资'`

**已推送：** commit `87c780a` → GitHub main 分支

## 优化：导出Excel支持按搜索结果和列设置过滤

**需求：**
- 导出时只导出当前搜索结果，而不是全部数据
- 用户在列设置中屏蔽的字段不导出

**后端修改：** `backend/app.py` → `export_excel()`
- 新增接收查询参数：`search`（全局搜索关键词）、`field_filters`（字段级筛选JSON）、`visible_fields`（可见字段列表JSON）
- `search`：遍历物资 custom_fields 值，匹配则保留
- `field_filters`：按字段名/值精确筛选
- `visible_fields`：非空时只导出列表中的字段；为空时导出所有字段（兼容旧行为）
- 始终排除图片字段

**前端修改：** `frontend/src/views/Inventory.vue` → `exportExcel()`
- 构建 `URLSearchParams`，传入 `searchKeyword` 和 `displayedFields`（列设置中未屏蔽的字段）
- 后端按这些参数过滤后导出

**已推送：** commit `64684fb` → GitHub main 分支

## 新增：重建ID功能

**需求：**
物资ID不连续（因历史删除导致空缺），希望在前端界面提供"重建ID"按钮，点击后ID从1连续显示；再次点击恢复原始ID。导出Excel时同步显示重建后的序号。

**实现（纯前端显示层，不修改数据库）：**

### 前端 `frontend/src/views/Inventory.vue`
- `data()` 新增 `rebuildIdMode: false`
- header-actions 区域新增切换按钮：默认文字"重建ID"，激活后变为"恢复原始ID"（警告色），带 tooltip
- 桌面端表格：ID列用 `scope.$index + (currentPage-1)*pageSize + 1` 代替 `scope.row.id`（仅在 rebuildIdMode 时）
- 移动端卡片：`item.id` 改为 `(currentPage-1)*pageSize + index + 1`（仅在 rebuildIdMode 时）
- `exportExcel()`：rebuildIdMode 为真时追加 `rebuild_id=1` 参数

### 后端 `backend/app.py` → `export_excel()`
- 新增参数 `rebuild_id`（默认 0）
- 构建导出数据时：`rebuild_id=1` → `export_id = seq`（枚举序号），否则 `export_id = mid`（原始ID）

**已推送：** commit `cf01687` → GitHub main 分支
